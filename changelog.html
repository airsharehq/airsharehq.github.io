Changelog

All notable changes to AirShare will be documented in this file.

The format is based on Keep a Changelog, and this project adheres to Semantic Versioning.

[0.1.1] - 2025-01-15

Changed

  • Modal Close Behavior: Completely redesigned modal close behavior for consistent user experience across all platforms
  • Windows Modal Handling: Optimized Windows-specific modal closing to prevent flickering and ghosting
  • Cross-Platform Consistency: Unified modal close logic across Windows, macOS, and Linux
  • Application Version: Updated to v0.1.1 for major UX improvements

Fixed

  • Modal Flickering: Eliminated "trznut" (flash) behavior on modal close (X button)
  • Windows Ghost Windows: Prevented Windows ghost modal artifacts with improved window management
  • Modal Animation: Removed unnecessary fade animations during close for instant hide behavior
  • Dark Mode Issues: Standardized light appearance across all platforms and modals
  • Close Handler: Streamlined close handler logic for all platform modal components
  • Visual Consistency: Fixed title bar and close button behavior for unified experience

Technical

  • BaseDialog Component: Major refactoring of src/ui/modals/base_dialog.py close logic
  • Platform Detection: Simplified platform-specific logic for modal closing
  • Animation Removal: Eliminated transparency-based fade animations on modal close
  • Memory Management: Enhanced modal cleanup to prevent visual artifacts and focus issues
  • Windows DWM: Optimized Windows Desktop Window Manager interactions for clean close handling

Fixed

Technical

Development Notes

Version Numbering

  • Major version: Breaking changes or major feature additions
  • Minor version: New features or significant improvements
  • Patch version: Bug fixes and minor improvements

Release Process

  1. Update version numbers in constants/app_info.py
  2. Update CHANGELOG.md with new features/fixes
  3. Test on all target platforms
  4. Create release build
  5. Test installation and basic functionality
  6. Publish release

Testing

  • Unit tests for core functionality
  • Integration tests for file transfers
  • Cross-platform compatibility testing
  • UI/UX testing on different screen sizes
  • Network connectivity testing
  • Error handling verification

  • Type hints where appropriate
  • Comprehensive error handling

  • Performance optimization
